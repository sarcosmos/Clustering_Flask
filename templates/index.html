Clustering de Clientes con K-Means

Se agruparon {{ data.total_clientes }} clientes del dataset Telco Customer Churn en {{ data.n_clusters }} clusters. La tabla de clientes muestra solo los primeros {{ data.clientes_mostrados }} registros para mantener la pagina ligera.

Descripcion del problema

El problema consiste en encontrar grupos de clientes con comportamientos similares sin usar una etiqueta previa. En lugar de predecir directamente si un cliente hara churn, el objetivo es segmentar la base de clientes para entender perfiles de uso, gasto, antiguedad y tipo de contrato.

Esta segmentacion puede ayudar a identificar clientes nuevos de alto riesgo, clientes estables, clientes con servicios premium y grupos que requieren estrategias comerciales diferentes.

Explicacion del dataset

El archivo utilizado es WA_Fn-UseC_-Telco-Customer-Churn.csv. Contiene informacion de clientes de una empresa de telecomunicaciones, incluyendo datos demograficos, servicios contratados, tipo de contrato, metodo de pago, cargos mensuales, cargos totales y la variable Churn.

Para el agrupamiento se excluyeron customerID, porque solo identifica al cliente, y Churn, porque es una etiqueta de resultado. Churn se conserva despues para interpretar que tan riesgoso es cada cluster.

Conceptos principales de K-Means

Cluster

Grupo de clientes que son parecidos entre si segun las variables usadas.

Centroide

Punto central de un cluster. Representa el perfil promedio del grupo.

Distancia

K-Means asigna cada cliente al centroide mas cercano.

Inercia

Suma de distancias internas. Mientras menor sea, mas compactos son los clusters.

Ejecucion del modelo

  1. Se lee el CSV desde la carpeta data.
  2. Se convierte TotalCharges a numero y se completan valores vacios con la mediana.
  3. Las variables categoricas se convierten a columnas numericas con codificacion one-hot.
  4. Las variables se escalan con StandardScaler para que ninguna domine por su magnitud.
  5. Se ejecuta KMeans con {{ data.n_clusters }} clusters, random_state=42 y n_init=10.

Despues de convertir categorias a variables numericas, el modelo utiliza {{ data.columnas_modelo|length }} columnas.

Interpretacion de resultados

Los clusters permiten comparar perfiles. Un cluster con mayor porcentaje de churn puede necesitar acciones de retencion; uno con mayor antiguedad y cargos altos puede representar clientes valiosos y estables.
